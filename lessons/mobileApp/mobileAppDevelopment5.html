Cookie clicker, REST API

V této lekci si vysvětlíme jak vytvořit jednoduchou hru, která se jmenuje Cookie clicker. Hra je jednoduchá, klikáte na tlačítko a získáváte body. Také si vysvětlíme co je to REST API a jak ho použít.

Vytvoření Cookie Clickeru v Thunkable

Vytvoříme jednoduchou hru Cookie Clicker v Thunkable, kde bude hráč sbírat body klikáním na obrázek sušenky. Tento projekt slouží k procvičení proměnných, událostí (when click) a podmínek (if...else).

Komponenty aplikace

Plocha 1 - main screen

  • Label: zobrazí aktuální počet bodů.
  • Label: zobrazí aktulání počet CPS (Cookie per second)
  • Button (btnCookie): obrázek sušenky, který hráč kliká pro získání bodů.

Plocha 2 - upgrades screen

  • Label: zobrazí aktuální počet bodů.
  • Label: informace o vylepšení auto clickeru
  • Button (btnCookie): Text bude cena za auto clicker
  • Label: informace o vylepšení auto pusher
  • Button (btnCookie): Text bude cena za auto pusher

Nejprve je nutné vytvořit dvě globální proměnné, které budou sloužit k uchování důležitých hodnot ve hře. Proměnná CPS (cookies per second) je nastavena na hodnotu 1, což znamená, že každou sekundu hráč získá jednu sušenku. Druhá proměnná cookies je nastavena na 0, protože hráč na začátku nemá žádné sušenky.

Když uživatel klikne na tlačítko cookie, spustí se funkce addCookies. Tato funkce zvýší počet sušenek (cookies) o aktuální hodnotu proměnné CPS. Zároveň se aktualizuje text na popisku (Label1), aby hráč viděl aktuální počet sušenek.

Aby se sušenky přičítaly i automaticky, je využit časovač s názvem CookieTimer. Tento časovač se spustí a v pravidelných intervalech volá funkci addCookies, čímž se sušenky přičítají i bez nutnosti klikání.

Když se otevře Screen1, je potřeba zobrazit aktuální počet sušenek a hodnotu CPS. To je zajištěno nastavením textových popisků ihned po načtení obrazovky. Díky tomu má hráč vždy přehled o svém aktuálním stavu.

V aplikaci je také tlačítko Upgrade, které slouží k otevření obrazovky Screen2. Kliknutím na toto tlačítko se hráč přesměruje na druhou obrazovku, kde pravděpodobně bude možné vylepšit hru, například zvýšit hodnotu CPS.



Při spuštění Screen2 se nastaví ceny a základní hodnoty upgradů, jako jsou autoclicker, autopusher, farm a factory. Každý upgrade má svou počáteční cenu a ovlivňuje rychlost získávání sušenek.

Po otevření Screen2 se zobrazí aktuální počet sušenek, aby hráč viděl svůj postup

Kliknutím na tlačítko autoclicker nebo autopusher se nejprve ověří, zda hráč má dostatek sušenek. Pokud ano, cena upgradu se odečte, zvýší se rychlost generování sušenek (CPS), cena upgradu se zvýší o 20 % a aktualizují se texty tlačítek a počtu sušenek.

Kliknutím na tlačítko back se hráč vrátí na Screen1, kde může pokračovat ve sbírání sušenek a využívat zakoupené upgrady.

Co je API, REST a JSON?

Co je API?

API (Application Programming Interface) je rozhraní, které umožňuje komunikaci mezi různými softwarem. Představte si, že sedíte u stolu v restauraci s nabídkou, ze které si můžete objednat. Kuchyně je „software“, který připraví vaši objednávku. Chybí však kritické spojení pro komunikaci vaší objednávky do kuchyně a doručení vašeho jídla zpět na váš stůl. Zde přichází na řadu číšník, který je vlastně API.

Příklad: Představte si, že si rezervujete let online. Při zadání informací o letu vlastně komunikujete s API letecké společnosti, které vám vrací dostupné lety a ceny.

Co je REST?

REST (Representational State Transfer) je soubor instrukcí, jak vytvořit efektivní, rychlá a spolehlivá webová API. RESTful API je přístupné prostřednictvím URL, používá HTTP metody a přenáší data ve formátu JSON.

Příklad: RESTful API se používá v mnoha aplikacích pro získávání dat, jako jsou například informace o počasí nebo letecké spoje.

Co je JSON?

JSON (JavaScript Object Notation) je formát pro výměnu dat. Používá se pro přenos dat mezi klientem a serverem. JSON popisuje objekty, abstraktní entity a jejich vztahy. Formát vypadá takto:

{
    "first_key": "first_value",
    "second_key": "second_value"
}

JSON může obsahovat i vnořené objekty:

{
    "emp_id": 120412,
    "name": "Smith Jones",
    "address": {
        "street": "Washington 24th Street",
        "city": "New York"
    }
}

HTTP Požadavky

HTTP (Hypertext Transfer Protocol) je protokol pro komunikaci mezi systémy na webu. Zde jsou základní HTTP metody:

  • GET – Načítá data ze serveru.
  • POST – Odesílá data na server.
  • PUT – Vytváří nebo nahrazuje zdroj.
  • DELETE – Odstraní zadaný prostředek.

Jak využíváme API, REST a JSON?

V praxi API, REST a JSON používáme pro získání a odesílání dat mezi aplikacemi a servery. Můžeme například použít API pro zobrazení informací o počasí, letech, nebo dokonce pro ovládání jiných zařízení. Díky těmto technologiím můžeme aplikace rychle propojit s databázemi a jinými online službami.
