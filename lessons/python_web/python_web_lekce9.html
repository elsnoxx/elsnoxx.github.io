Co je to MVC a proč ho používat?

  • MVC znamená Model-View-Controller – je to architektonický vzor pro návrh aplikací.
  • Model – stará se o data a logiku aplikace (např. práce s databází).
  • View – zobrazuje data uživateli (HTML šablony, stránky).
  • Controller – zpracovává požadavky uživatele, propojuje model a view.

MVC pomáhá oddělit jednotlivé části aplikace, což usnadňuje údržbu a rozšiřování kódu.

Design Patterns, MVC a MVT

Co jsou design patterns?

Design patterns (návrhové vzory) jsou osvědčené postupy, jak řešit běžné problémy při vývoji softwaru. Pomáhají psát přehledný, udržitelný a znovupoužitelný kód. Stejně jako stavební plány nebo recepty, i návrhové vzory dávají vývojářům jasný návod, jak postupovat.

MVC a MVT

  • MVC (Model-View-Controller) je architektonický vzor, který rozděluje aplikaci na tři části:
    • Model – stará se o data a logiku aplikace (např. práce s databází).
    • View – zobrazuje data uživateli (HTML šablony, stránky).
    • Controller – zpracovává požadavky uživatele, propojuje model a view.
  • MVT (Model-View-Template) je varianta MVC používaná v Djangu:
    • Template – šablona, která určuje, jak budou data zobrazena uživateli.

Hlavní výhodou těchto vzorů je oddělení jednotlivých částí aplikace, což usnadňuje údržbu, testování a rozšiřování kódu.

Proč je to důležité?

  • Umožňuje více lidem pracovat na jedné aplikaci (každý na jiné části).
  • Zjednodušuje úpravy a rozšiřování aplikace.
  • Zvyšuje přehlednost a kvalitu kódu.

V této lekci si ukážeme, jak MVC/MVT funguje v praxi na jednoduchých příkladech v Djangu.

Jak funguje MVC v praxi?

  • Uživatel odešle požadavek (např. klikne na tlačítko).
  • Controller přijme požadavek, zpracuje ho a požádá Model o data.
  • Model vrátí data Controlleru.
  • Controller předá data View, které je zobrazí uživateli.

MVC v Pythonu pomocí Django

  • Django je populární Python framework, který využívá principy MVC (v Djangu se často označuje jako MVT – Model-View-Template).
  • Model – třídy v models.py, které popisují data (např. tabulky v databázi).
  • View – funkce nebo třídy v views.py, které zpracovávají požadavky a vrací odpovědi.
  • Template – HTML šablony, které zobrazují data uživateli.

Příklad jednoduchého View v Djangu:

# views.py
from django.http import HttpResponse

def hello(request):
    return HttpResponse("Ahoj světe!")

Vytvoření a spuštění první Python (Django) aplikace

Oficiální dokumentace

Vyzkoušej si vytvořit a spustit jednoduchou Django aplikaci pomocí pip a příkazové řádky. Postupuj podle těchto kroků:

  1. Nainstaluj Django pomocí pip: 
    pip install django
  2. Vytvoř nový Django projekt: 
    django-admin startproject mujprojekt
  3. Přejdi do složky projektu: 
    cd mujprojekt
  4. Spusť vývojový server: 
    python manage.py runserver
  5. Otevři prohlížeč a zadej adresu: http://127.0.0.1:8000/ a uvidíš úvodní obrazovku Django projektu:
  6. Můžeš vytvořit aplikaci v rámci projektu: 
    python manage.py startapp mojeaplikace
    Aplikaci pak přidej do INSTALLED_APPS v souboru settings.py.

Pokud narazíš na problém, podívej se do oficiální dokumentace Django nebo se zeptej lektora.

Struktura adresářů v Django projektu

Po vytvoření nového Django projektu a aplikace bude adresářová struktura vypadat například takto:

  • manage.py – Hlavní skript pro správu projektu (spouštění serveru, migrace databáze, spouštění aplikací).
  • db.sqlite3 – Výchozí databázový soubor (pokud používáš SQLite).
  • mujprojekt/ – Hlavní složka projektu, obsahuje konfiguraci projektu:
    • __init__.py – Označuje složku jako Python balíček.
    • settings.py – Hlavní konfigurační soubor projektu (nastavení databáze, aplikací, jazyků, atd.).
    • urls.py – Definuje URL adresy a směrování požadavků v rámci projektu.
    • wsgi.py – Slouží ke spuštění projektu na serveru (produkční nasazení, WSGI).
    • asgi.py – Slouží pro asynchronní nasazení (např. WebSockety, ASGI).
  • mojeaplikace/ – Složka tvé aplikace (každá aplikace má vlastní složku):
    • __init__.py – Označuje složku jako Python balíček.
    • admin.py – Nastavení administrace pro tuto aplikaci (co bude vidět v Django adminu).
    • apps.py – Konfigurace aplikace (název, nastavení).
    • models.py – Definice datových modelů (tabulky v databázi).
    • views.py – Funkce nebo třídy, které zpracovávají požadavky a vrací odpovědi (logika aplikace).
    • tests.py – Testy pro tuto aplikaci.
    • migrations/ – Složka s migracemi databáze (změny v modelech).
      • __init__.py – Označuje složku jako Python balíček.
      • další soubory – Jednotlivé migrace (např. 0001_initial.py).

Tip: Nejčastěji budeš pracovat se soubory settings.py, urls.py (v hlavním projektu), a v aplikaci models.py, views.py, případně admin.py a složkou migrations/.

Jak vytvořit úvodní stránku v Django aplikaci

  1. Vytvoř view (funkci) pro úvodní stránku: 
    # mojeaplikace/views.py
from django.shortcuts import render

def home(request):
    return render(request, 'home.html')
  2. Vytvoř šablonu (template):
    • V adresáři mojeaplikace vytvoř složku templates (pokud ji nemáš).
    • Do ní vlož soubor home.html:
    <!-- mojeaplikace/templates/home.html -->
<h1>Vítej na úvodní stránce!</h1>
<p>Toto je tvoje první Django stránka.</p>
  3. Zaregistruj URL adresu:
    • V souboru mojeaplikace/urls.py (pokud neexistuje, vytvoř ho):
    # mojeaplikace/urls.py
from django.urls import path
from . import views

urlpatterns = [
    path('', views.home, name='home'),
]
    • V hlavním urls.py projektu přidej odkaz na mojeaplikace:
    # mujprojekt/urls.py
from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', include('mojeaplikace.urls')),  # úvodní stránka bude na /
]
  4. Spusť server a otevři http://127.0.0.1:8000/
    • Měla by se zobrazit tvoje úvodní stránka!

Tip: Pokud šablona není nalezena, zkontroluj, že máš správně nastavené TEMPLATES v settings.py a že složka templates je ve správném adresáři.

Nejjednodušší Django projekt bez databáze – krok za krokem

Pokud si chceš rychle vyzkoušet, jak Django funguje bez práce s databází, můžeš vytvořit úplně jednoduchý projekt, který zobrazí jen úvodní stránku. Tento postup ti ukáže základní principy práce s view, URL a šablonou.

  1. Vytvoř nový projekt a aplikaci: 
    django-admin startproject mujprojekt
cd mujprojekt
python manage.py startapp mojeaplikace
  2. Přidej aplikaci do INSTALLED_APPS v settings.py: 
    INSTALLED_APPS = [
    # ... ostatní aplikace ...
    'mojeaplikace',
]
  3. Vytvoř view pro úvodní stránku: 
    # mojeaplikace/views.py
from django.shortcuts import render

def home(request):
    return render(request, 'home.html')
  4. Vytvoř šablonu:
    • Vytvoř složku mojeaplikace/templates/ a do ní soubor home.html:
    <!-- mojeaplikace/templates/home.html -->
<h1>Vítej na stránce!</h1>
<p>Toto je tvoje první Django stránka bez databáze.</p>
  5. Vytvoř urls.py v aplikaci: 
    # mojeaplikace/urls.py
from django.urls import path
from . import views

urlpatterns = [
    path('', views.home, name='home'),
]
  6. Propoj aplikaci v hlavním urls.py projektu: 
    # mujprojekt/urls.py
from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', include('mojeaplikace.urls')),
]
  7. Spusť server a otevři http://127.0.0.1:8000/: 
    python manage.py runserver
    • Uvidíš jednoduchou úvodní stránku bez databáze.

Tip: Tento miniprojekt je ideální pro pochopení základní struktury Django aplikace bez zbytečných komplikací.
