ToDo aplikace v Djangu

V této lekci se si vyzkoušíme předělat námi vztvořenou aplikace ve Flasku do Djanga.

Nejprve si ukážeme jak vypadala nase aplikace ve Flasku, kterou jsme si vytvořili v předchozí lekci. A potom se vrhneme na přepis do Djanga.

Úvod - Převod Flask TODO aplikace do Djanga

V předchozí lekci jsme si vytvořili TODO aplikaci ve Flasku. Nyní si ukážeme, jak tutéž aplikaci přepsat do Djanga s využitím databáze místo seznamů v paměti.

Co naše TODO aplikace umí:
  • Přidávání nových úkolů s popisem, poznámkou, očekávaným termínem a předmětem
  • Označování úkolů jako splněných
  • Mazání úkolů
  • Editace existujících úkolů
  • Zobrazení historie splněných a smazaných úkolů
Výhoda Djanga: Data se budou ukládat do databáze, takže se neztratí po restartu serveru!

Krok 1: Inicializace Django projektu

  1. Vytvoření projektu a aplikace: 
    
django-admin startproject todo_projekt
python manage.py startapp todo_app
cd todo_projekt
python manage.py runserver
  2. Přidání aplikace do settings.py: 
    INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    'todo_app',  # naše aplikace
]
  3. Vytvoření složek pro šablony a statické soubory: 
    mkdir todo_app/templates
todo_app/static
todo_app/static/css
todo_app/static/js

Krok 2: Vytvoření modelů a jejich migrace

Model pro TODO úkoly:
# todo_app/models.py
from django.db import models
from django.utils import timezone

class Todo(models.Model):
    # text => charField, maximalní délka 200 znaků, nazev "Úkol"
    text = NULL
    # poznamka => textField, volitelné, nazev "Poznámka"
    poznamka = NULL
    # ocekavane => dateField, volitelné, nazev "Očekávaný
    ocekavane = NULL
    # predmet => charField, maximalní délka 100 znaků, volitelné, nazev "Předmět"
    predmet = NULL
    # done => booleanField, výchozí hodnota False, nazev "Splněno"
    done = NULL
    # created => dateTimeField, výchozí hodnota aktuální čas, nazev "Vytvořeno"
    created = NULL
    # completed => dateTimeField, volitelné, nazev "Dokončeno"
    completed = NULL
    # deleted => booleanField, výchozí hodnota False, nazev "Smazáno"
    deleted = NULL
    # deleted_at => dateTimeField, volitelné, nazev "Smazáno v"
    deleted_at = NULL
    
    # Přidání metody pro zobrazení úkolu __str__
    def __str__(self):
Spuštění migrací:
python manage.py makemigrations todo_app
python manage.py migrate
Vysvětlení polí:
- text: Hlavní popis úkolu
- poznamka: Volitelná poznámka
- ocekavane: Termín splnění
- predmet: Kategorie/předmět úkolu
- done: Zda je úkol splněn
- deleted: Soft delete - úkol není skutečně smazán

Krok 3: Vytvoření superuživatele a admin rozhraní

  1. Vytvoření superuživatele: 
    python manage.py createsuperuser
    Django se zeptá na uživatelské jméno, email a heslo. Heslo musí být dostatečně silné!
  2. Registrace modelu v admin rozhraní: 
    # todo_app/admin.py
from django.contrib import admin
from .models import Todo

@admin.register(Todo)
class TodoAdmin(admin.ModelAdmin):
    list_display = ['text', 'predmet', 'ocekavane', 'done', 'created']
    list_filter = ['done', 'deleted', 'predmet', 'created']
    search_fields = ['text', 'poznamka']
    list_editable = ['done']
    date_hierarchy = 'created'
  3. Spuštění serveru a přístup k adminu: 
    python manage.py runserver

    Přejdi na http://127.0.0.1:8000/admin/ a přihlas se svými údaji.

Vysvětlení jednotlivých views:
📝 index(request)
  • GET: Zobrazí všechny nesmazané úkoly
  • POST: Přidá nový úkol do databáze
  • Filtr: Todo.objects.filter(deleted=False)
  • Řazení: .order_by('-created')
✅ done(request, pk)
  • Toggle: todo.done = not todo.done
  • Čas: todo.completed = timezone.now()
  • Bezpečnost: get_object_or_404(Todo, pk=pk)
  • Přesměrování: return redirect('index')
🗑️ delete(request, pk)
  • Soft delete: todo.deleted = True
  • Čas smazání: todo.deleted_at = timezone.now()
  • Zachování dat pro historii
  • Bezpečnější než skutečné mazání
✏️ edit(request, pk)
  • GET: render(request, 'edit.html', {'todo': todo})
  • POST: Aktualizace všech polí
  • Formulář předvyplněný aktuálními daty
  • Validace a uložení změn
📚 history(request)
  • Splněné: Todo.objects.filter(done=True, deleted=False)
  • Smazané: Todo.objects.filter(deleted=True)
  • Řazení podle času: .order_by('-completed')
  • Dva seznamy v jednom template
Klíčové Django funkce:
  • get_object_or_404(Model, pk=pk): Bezpečně načte objekt
  • request.method == 'POST': Kontrola typu požadavku
  • request.POST.get('field'): Získání dat z formuláře
  • redirect('url_name'): Přesměrování
  • render(request, 'template.html', context): Vykreslení
