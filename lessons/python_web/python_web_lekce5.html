API, REST, JSON

V této lekci se naučíme, co je API, REST a JSON. Tyto technologie jsou klíčové pro moderní webový vývoj a umožňují efektivní komunikaci mezi aplikacemi a servery. Pochopení těchto konceptů nám pomůže lépe porozumět tomu, jak fungují webové aplikace a jak můžeme efektivně pracovat s daty.

Co je API, REST a JSON?

Co je API?

API (Application Programming Interface) je rozhraní, které umožňuje komunikaci mezi různými softwarem. Představte si, že sedíte u stolu v restauraci s nabídkou, ze které si můžete objednat. Kuchyně je „software“, který připraví vaši objednávku. Chybí však kritické spojení pro komunikaci vaší objednávky do kuchyně a doručení vašeho jídla zpět na váš stůl. Zde přichází na řadu číšník, který je vlastně API.

Příklad: Představte si, že si rezervujete let online. Při zadání informací o letu vlastně komunikujete s API letecké společnosti, které vám vrací dostupné lety a ceny.

Co je REST?

REST (Representational State Transfer) je soubor instrukcí, jak vytvořit efektivní, rychlá a spolehlivá webová API. RESTful API je přístupné prostřednictvím URL, používá HTTP metody a přenáší data ve formátu JSON.

Příklad: RESTful API se používá v mnoha aplikacích pro získávání dat, jako jsou například informace o počasí nebo letecké spoje.

Co je JSON?

JSON (JavaScript Object Notation) je formát pro výměnu dat. Používá se pro přenos dat mezi klientem a serverem. JSON popisuje objekty, abstraktní entity a jejich vztahy. Formát vypadá takto:

{
    "first_key": "first_value",
    "second_key": "second_value"
}

JSON může obsahovat i vnořené objekty:

{
    "emp_id": 120412,
    "name": "Smith Jones",
    "address": {
        "street": "Washington 24th Street",
        "city": "New York"
    }
}

HTTP Požadavky

HTTP (Hypertext Transfer Protocol) je protokol pro komunikaci mezi systémy na webu. Zde jsou základní HTTP metody:

  • GET – Načítá data ze serveru.
  • POST – Odesílá data na server.
  • PUT – Vytváří nebo nahrazuje zdroj.
  • DELETE – Odstraní zadaný prostředek.

Jak funguje komunikace pomocí API?

Krok za krokem - proces komunikace

  1. Klient zahájí požadavek - Uživatel nebo aplikace zahájí akci (např. kliknutí na tlačítko "Zobrazit počasí")
  2. Vytvoření HTTP požadavku - Aplikace vytvoří HTTP požadavek s URL, metodou (GET/POST) a případnými parametry
  3. Odeslání požadavku na server - Požadavek se odešle na server API přes internet
  4. Server zpracuje požadavek - API server přijme požadavek, ověří klíč/autorizaci a zpracuje jej
  5. Server vytvoří odpověď - Server připraví požadovaná data ve formátu JSON
  6. Odpověď se vrátí klientovi - Data putují zpět ke klientovi
  7. Klient zpracuje odpověď - Aplikace analyzuje JSON data a použije je (např. zobrazí teplotu)

Analogie z reálného života

Představte si celý proces jako návštěvu restaurace:

  1. Vy (klient) si prohlížíte menu (dostupná API)
  2. Vytvoříte objednávku (HTTP požadavek) a sdělíte ji číšníkovi
  3. Číšník předá objednávku do kuchyně (server API)
  4. Kuchyně objednávku zpracuje podle dostupných surovin
  5. Kuchyně připraví jídlo a předá ho číšníkovi (JSON odpověď)
  6. Číšník vám doručí jídlo (přenos dat zpět ke klientovi)
  7. Vy si můžete jídlo vychutnat (zpracování odpovědi)

Typické scénáře komunikace

Scénář HTTP metoda Popis procesu
Získání dat o počasí GET Klient pošle požadavek s parametry (město, API klíč) → Server vrátí aktuální data o počasí v JSON
Přihlášení uživatele POST Klient pošle přihlašovací údaje → Server ověří údaje a vrátí autentizační token v JSON
Aktualizace profilu PUT Klient pošle aktualizované údaje profilu → Server aktualizuje profil a vrátí potvrzení
Smazání příspěvku DELETE Klient pošle ID příspěvku ke smazání → Server smaže příspěvek a vrátí potvrzení

Praktická ukázka komunikace s API v Pythonu

Zde je jednoduchý příklad, který demonstruje komunikaci s API pomocí knihovny requests. V tomto případě získáváme informace o ekonomickém subjektu z registru ARES:


import requests

# 1. Definujeme URL API koncového bodu
url = "https://ares.gov.cz/ekonomicke-subjekty-v-be/rest/ekonomicke-subjekty/61989100"

# 2. Odešleme GET požadavek na server
print("Odesílám požadavek na API...")
response = requests.get(url)

# 3. Zkontrolujeme, zda byl požadavek úspěšný (stavový kód 200)
if response.status_code == 200:
    # 4. Převedeme JSON odpověď na slovník Pythonu
    data = response.json()
    
    # 5. Vypíšeme základní informace o ekonomickém subjektu
    print("\nZÁKLADNÍ INFORMACE O SUBJEKTU:")
    print(f"Název: {data['nazev']}")
    print(f"IČO: {data['ico']}")
    print(f"Právní forma: {data['pravniForma']['nazev']}")
    print(f"Adresa: {data['sidlo']['textovaAdresa']}")
else:
    print(f"Chyba při komunikaci s API. Stavový kód: {response.status_code}")

Co tento kód dělá?

  1. Importujeme knihovnu requests pro HTTP požadavky
  2. Definujeme URL koncového bodu API s konkrétním IČO
  3. Odesíláme GET požadavek a ukládáme odpověď
  4. Kontrolujeme úspěšnost požadavku pomocí stavového kódu
  5. Převádíme JSON odpověď na datovou strukturu Pythonu
  6. Zobrazujeme vybrané informace o subjektu

Jak využíváme API, REST a JSON?

V praxi API, REST a JSON používáme pro získání a odesílání dat mezi aplikacemi a servery. Můžeme například použít API pro zobrazení informací o počasí, letech, nebo dokonce pro ovládání jiných zařízení. Díky těmto technologiím můžeme aplikace rychle propojit s databázemi a jinými online službami.

Práce s JSON soubory

JSON (JavaScript Object Notation) je lehký formát pro výměnu dat. V Pythonu můžeme JSON snadno číst a zapisovat pomocí modulu json.

1. Čtení JSON souboru


import json

# Načtení JSON souboru
with open('data.json', 'r') as file:
    data = json.load(file)

# Přístup k datům
print(data['name'])
print(data['address']['city'])

2. Zápis do JSON souboru


import json

# Data k uložení
data = {
    "name": "John Doe",
    "age": 30,
    "address": {
        "street": "123 Main St",
        "city": "New York"
    }
}

# Zápis do JSON souboru
with open('data.json', 'w') as file:
    json.dump(data, file, indent=4)

JSON soubory jsou užitečné pro ukládání konfigurací, dat aplikací nebo pro výměnu dat mezi klientem a serverem.

Práce s JSON a HTTP odpověďmi pomocí knihovny requests

Knihovna requests umožňuje snadnou práci s HTTP požadavky. Můžeme ji použít pro získání dat ve formátu JSON z API.

1. Základní GET požadavek


import requests

# Odeslání GET požadavku
response = requests.get('https://jsonplaceholder.typicode.com/posts/1')

# Získání dat ve formátu JSON
data = response.json()

# Přístup k datům
print(data['title'])
print(data['body'])

2. Odeslání POST požadavku


import requests

# Data k odeslání
payload = {
    "title": "foo",
    "body": "bar",
    "userId": 1
}

# Odeslání POST požadavku
response = requests.post('https://jsonplaceholder.typicode.com/posts', json=payload)

# Získání odpovědi
print(response.status_code)
print(response.json())

Tato metoda je užitečná pro komunikaci s API, například při odesílání dat na server nebo získávání dat z API.

Ukázka práce s Weather API

Weather API poskytuje informace o aktuálním počasí. Pomocí knihovny requests můžeme získat data o počasí pro konkrétní město.

1. Získání aktuálního počasí


import requests

# URL API a klíč
url = 'https://api.weatherapi.com/v1/current.json'
params = {
    'key': '939716ed2d204c008ef151039252603',
    'q': 'Prague'
}

# Odeslání GET požadavku
response = requests.get(url, params=params)

# Získání dat ve formátu JSON
data = response.json()

# Výpis informací o počasí
print(f"Město: {data['location']['name']}")
print(f"Teplota: {data['current']['temp_c']} °C")
print(f"Stav: {data['current']['condition']['text']}")

Tento příklad ukazuje, jak získat aktuální informace o počasí pro město Praha pomocí Weather API.
